# 极简 AI 语音记账 —— 完全免费技术方案 & 需求文档

## Agent / 自动化（Harness）

协作入口与约束见 [AGENTS.md](./AGENTS.md)；分阶段可勾选验证清单见 [docs/AGENT_CHECKLIST.md](./docs/AGENT_CHECKLIST.md)。本地环境备忘见 [docs/本地开发环境.md](./docs/本地开发环境.md)。环境变量与安全分层见 [docs/环境变量与密钥.md](./docs/环境变量与密钥.md)，模板见 [`.env.example`](./.env.example)。本地执行 `npm install` 后运行 `npm test`；持续集成见仓库 [.github/workflows/ci.yml](./.github/workflows/ci.yml)（工作流名 `ci`）。

目标：快速搭建一个个人可用、无广告、极简的网页语音记账工具。技术尽量使用免费/浏览器原生工具，能在几天内做出可用版本。

核心功能（4 步）
1. 打开网页
2. 按住说话（或点击开始）录音，浏览器把语音转成文字（Web Speech API）
3. 把识别出的文字发给大模型做结构化解析（提取：金额、商家、分类、时间）
4. 把解析结果经 **Serverless API** 写入 **Neon（Postgres）**，显示在今天的账单列表/统计中

非目标（短期）
- 多用户注册/复杂权限（只做单用户或简单 API key）
- 发票扫描、银行对账、复杂财务报表
- 离线模型推理（先用云 API）

一、技术栈（全部可免费）
- 前端：纯 HTML + CSS + 原生 JS（或可选 Flutter Web）
- 语音转文字（STT）：浏览器原生 Web Speech API（免费）
- AI 解析：任意支持 HTTP 的大模型 API（OpenAI、讯飞、通义等），用量小成本低
- 后端/数据库：**Neon**（免费档 Serverless Postgres，个人轻量足够）+ **Vercel Serverless**（或 Netlify Function、Cloudflare Workers 等）提供 HTTP API；**不在浏览器中存放数据库连接串**
- 部署：GitHub Pages / Vercel（免费）

**可选替代（一句）**：若已押注 Cloudflare Pages，可用 **Cloudflare D1 + Worker** 同属免费生态，但 SQL 需按 SQLite 方言微调，与下文第四节 Postgres 示例不完全一致。

二、用户故事（最小可发布版本）
- 作为用户，我可以按住一个按钮说话并保存一条账单。
- 作为用户，我可以在首页看到“今日总支出”。
- 作为用户，我可以查看按天的账单列表。
- 作为用户，我可以查看本月按分类的饼图/统计。

三、最小系统设计（前后端交互）
1. 前端：
   - 录音并把结果转成文字（Web Speech API）
   - 将文字通过 fetch 发送给 AI 解析端点（可直接调用第三方模型 API，或经 serverless 代理）
   - 收到 JSON 后，通过 `fetch` 调用自建 API（如 `POST /api/expenses`）插入一条账单
   - 列表与统计页通过 `GET /api/expenses?from=&to=`（或等价查询参数）拉取数据，**不**从浏览器直连数据库
2. 数据库（Neon Postgres）：
   - 表：users（可选）、expenses、categories（预定义）；库表仅由 Serverless 通过服务端连接串访问
3. 显示：
   - 首页（大说话按钮 + 今日总额 + 最近几条）
   - 列表页（按天分组）
   - 统计页（本月饼图，分类汇总）

五、AI Prompt（严格只返回 JSON）
使用下面的指令把识别出的文字丢给模型，要求模型只输出 JSON，且字段固定。

Prompt（可直接使用）：
你是一个记账助手。请严格只输出一个 JSON 对象，不要包含任何解释或其他文本。  
从下面的用户话语中提取信息，返回字段如下：  
- amount: 数字（人民币金额，整数或小数），如果无法识别或无金额则为 null。  
- shop: 商家名称字符串，若无则为空字符串。  
- category: 只能选下面之一：餐饮、交通、购物、娱乐、日用、医疗、住房、其他。若无法判定，请返回 "其他"。  
- time: ISO 日期字符串，格式 YYYY-MM-DD（对话里若无明确日期，默认为“今天”的日期）。  
- note: 可选备注文本（如商品名、简短描述），若无则为空字符串。

async function saveExpense(parsed) {
  const res = await fetch('/api/expenses', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      // 个人项目可用简单 Bearer（与 API 环境变量中的密钥一致），勿把 Neon 连接串放前端
      'Authorization': 'Bearer ' + APP_API_TOKEN
    },
    body: JSON.stringify({
      amount: parsed.amount,
      shop: parsed.shop,
      category: parsed.category,
      occurred_at: parsed.time,
      note: parsed.note
    })
  });
  if (!res.ok) throw new Error('save failed');
  return res.json();
}

七、前端细节与 UX 建议
- 主按钮：按住说话（按下 start、松开 stop）；兼容没有按住功能的设备也可以点击切换“开始/停止”。
- 显示即时识别结果（文字），并在 AI 返回前使用“解析中”提示。
- 解析结果弹窗允许用户微调（可选），或直接快速保存（默认直接保存以保证极简流程）。
- 提供“撤销”或“编辑”最近一条的入口（防止误识别造成数据错误）。

八、部署与权限
- 将前端部署到 GitHub Pages 或 Vercel（静态站点）；**推荐**与 API **同域部署**（例如整站放在 Vercel：静态页 + `/api/*`），前端使用相对路径 `fetch('/api/expenses')`，省去跨域配置。
- **Neon**：在控制台创建项目，把 **连接串** 只配置在 Serverless 环境变量（如 `DATABASE_URL`）中，**永远不要**写进前端构建产物或公开仓库。
- **若静态页在 GitHub Pages、API 在另一域名**：必须在 API 侧正确配置 **CORS**（允许你的 Pages 域名），并仍只用 HTTPS + 简单鉴权保护写接口。
- AI Key：不要把高权限 key 写在前端。两种方法：
  1) 可接受风险的个人项目：把 AI key 放前端（不推荐，便捷）。
  2) 推荐：用一个轻量的 serverless 函数（Vercel Serverless / Netlify Function / Cloudflare Workers）作为代理，安全存放 AI KEY 并限制来源（CORS），该函数接收文本并调用 AI，再返回解析 JSON。可与 `/api/expenses` 放在**同一项目**（例如 `/api/parse`），成本可忽略。

九、隐私与安全
- 语音与文本将被发送到第三方模型，账单数据经你的 **Serverless API** 写入 **Neon**；明确只有个人账户使用；若敏感，请谨慎。
- 若不想把音频/文字发到模型，可只发送文本或在本地允许用户编辑后再发送。
- 鉴权与访问控制放在 **API 层**（如固定 Bearer token、校验来源、限流）；若未来多人使用，需单独设计登录与权限，勿依赖浏览器直连数据库。
