# 本地开发环境（凝练备忘）

本文记录本仓库在 **macOS** 上从零到能跑 **`npm test`** 的路径与易混点，供人与 Agent 复用。

## 1. 要装什么、装在哪

| 层级 | 是什么 | 装在哪 |
|------|--------|--------|
| **Node / npm** | 运行 JS、执行 `npm` 命令 | **整台电脑一份**（Homebrew 通常在 `/opt/homebrew/bin`，与当前终端所在**文件夹无关**） |
| **项目依赖** | 如 `vitest` | 各项目自己的 **`node_modules/`**（在仓库根执行 `npm install`） |

结论：**在哪个目录执行 `brew install node` 都一样**；换别的项目**不必**再装 Node，只需在新项目里各自 **`npm install`**。

## 2. 最短命令序列（本仓库）

```bash
cd /path/to/VoiceBillRecord
npm install
npm test
```

- `npm test` 通过即表示：Node、依赖、文档/脚手架文件齐全。

## 3. 若提示 `command not found: npm`

1. 安装 Node： **`brew install node`** 或 [nodejs.org](https://nodejs.org/) LTS。  
2. **PATH**：Apple Silicon 常见需在 `~/.zshrc` 中有 `export PATH="/opt/homebrew/bin:$PATH"`，保存后 **`source ~/.zshrc`** 或新开终端。  
3. 再执行第 2 节命令。

## 4. `brew install` 很慢或 `curl github.com` 超时

- Homebrew 会拉 **大体积依赖**（如 `icu4c`）或走 **ghcr.io**，国内直连 GitHub 常慢或超时。  
- **对策**：配置 **清华 Homebrew / Bottles** 与 **npmmirror（npm）**，见 [国内镜像配置.md](./国内镜像配置.md)。改 `~/.zshrc` / `~/.npmrc` 后需重新加载 shell。  
- `brew` 与 **`npm install` 可并行**：另开终端处理文档、`.env` 不影响安装进程（勿误关正在跑的 `brew` 终端）。

## 5. CI 与 `package-lock.json`

- 当前 GitHub Actions 使用 **`npm install`**（仓库可无 lockfile）。  
- 本地执行 `npm install` 若生成 **`package-lock.json`**，提交后可把 CI 改为 **`npm ci` + `cache: npm`** 以提速、锁版本（见 `.github/workflows/ci.yml` 内注释）。

## 6. 与 `.env` 的关系

- **镜像**：只影响 **brew / npm 下载**，与业务密钥无关。  
- **Neon / 大模型 Key**：见 [环境变量与密钥.md](./环境变量与密钥.md) 与根目录 `.env.example`；**勿**把 `DATABASE_URL` 等写进前端。
